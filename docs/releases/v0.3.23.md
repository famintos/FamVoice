### Changed

- Tightened the GitHub quality gates so pull requests and tagged releases now rerun the same frontend, Rust, and security checks before shipping.
- Expanded repository security scanning to cover the Rust backend in addition to the frontend workflows.
- Switched local secret and history persistence on Windows to use encrypted on-disk protection instead of plain-text fallbacks.

### Fixed

- Restored the CI security audit so Dependabot and normal pull requests no longer fail on an outdated `cargo-audit` invocation.
- Updated the Rust dependency lockfile to remove the blocked `rustls-webpki` advisory chain that was failing security checks.
- Aligned app version metadata across the package, Tauri config, and Rust crate so release tags now validate cleanly.
- Hardened frontend linting and cleaned up the current React warnings so the quality gate fails on real issues instead of warning-only drift.
