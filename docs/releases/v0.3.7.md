### Fixed

- Fixed a startup issue where FamVoice could open a broken widget page if Windows launched a development build without the local dev server running.
- Fixed unsafe Launch on Startup registrations from local debug or release builds so they no longer keep reopening the app into a broken state.

### Added

- Added a safety check that detects whether FamVoice is running from an installed app before allowing Launch on Startup.

### Changed

- Changed the Settings screen to disable Launch on Startup for non-installed builds and explain why the option is unavailable.
- Changed the GitHub release workflow to publish updater releases automatically with versioned release notes.
